서버 시작 -> web.xml / @WebServlet("요청 uri") 가장 먼저 읽어짐.

 web.xml - 매핑, 에러 페이지 작성, 지정 / welcome file : 설정 정보

	클라이언트 요청 대기
	요청 발생: 리퀘스트, 리스폰스 객체 생성 / 세션 기본 객체 생성(클라이언트의 첫 접속일 경우)
	url 분석 => 요청 uri(경로 - 컨텍스트 루트)
	요청에 맞는 방식에 맞추어 do get() or do post() 메서드 실행
	요청에 맞는 servlet이 없다면 path 경로 변경
	url path에 맞춰 경로에 맞는 파일 실행

	request: 요청 정보, URL, 클라이언트 ip(getRemoteAddress) 등
		요청 파라미터를 읽어 옴

	영역/속성 - 모든 xml문서는 4가지 속성에 속한다.
	page - pagecontext 매번 형성
	request - 하나의 요청에 실행되는 여러 개 페이지들이 하나의 값 공유
	세션: 새로운 도메인으로 클라이언트가 접속할 때 형성

	쿠키, 세션
	객체.setAttribute("이름", 값(object))

* EL
 - ${속성명}: 어플리케이션에 저장되어 있는 속성에 대한 Mapping 값을 가져 온다.
 - ${map.key}
 - ${일반객체.변수명}: 이 변수는 getter 혹은 is를 가지고 있어야 한다.
 - ${일반객체.메서드()}: 메서드가 반환한 값을 입력한다.
 -- ${list/arrar.인덱스(정수)} => 오류 발생하여 for each 사용 --
 - ${param.파라미터이름}: 해당 파라미터 이름으로 등록된 파라미터 값을 반환.
 
 - <%= null %>: null 출력
   ${null}: 빈 문자열 출력
   
 - ${test == null}과 같이 연산자 역시 el 부호 안에 적어야 한다.
